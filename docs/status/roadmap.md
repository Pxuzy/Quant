# 路线图和当前状态

## 当前状态摘要

当前项目已经具备数据可信后台的核心骨架，并开始具备股票研究台的基础入口：

- FastAPI 后端路由已覆盖健康检查、股票、行情、数据源、同步任务、数据集、数据质量、交易日历和数据库管理。
- 元数据模型已覆盖股票、数据源、同步任务、日志、定时配置、入库批次、数据集、交易日历和质量报告。
- Worker 已支持股票池、单股日线、市场级日线缺口补齐和交易日历任务。
- 前端已具备总控台、新闻汇总、数值数据、股票池、数据源管理、同步调度和数据库管理页面。
- 数据源 registry 已注册 AKShare、BaoStock、AData、Tushare、Stock SDK。
- 日线数据已按 Parquet 数据湖和 DuckDB 查询方向建设。
- 日线覆盖和市场补齐已按最近半年窗口管理。

主要差距不在底层模块数量，而在产品形态：研究台还不够像每天会用的股票工作台，新闻和自选股还没有形成闭环，部分页面仍偏后台卡片堆叠。

## P2 研究数据扩展

| 方向 | 说明 |
| --- | --- |
| 常用指标 | 均线、涨跌幅、成交额、换手等基础指标 |
| 财务和行业数据 | 在股票详情和筛选中逐步接入，不抢当前阶段 |
| 研究数据读取契约 | 先定义 `BarReader` 或 `DataPortal`，读取已治理日线、交易日历和后续指标 |
| 因子库 | 等读取契约、行情和新闻闭环稳定后再设计 |
| 回测输入接口 | 基于研究数据读取契约，不直接访问 provider、数据库表或 Parquet 物理路径 |
| 数据版本管理 | dataset version、manifest、分区清单、回滚策略 |

## P3 回测和 AI

| 方向 | 说明 |
| --- | --- |
| 最小回测引擎 | 先支持日线级、单市场、无实盘依赖的离线回测 |
| 策略配置 | 使用声明式配置描述股票池、时间范围、调仓频率、手续费和基准 |
| AI 研究助手 | 基于已治理数据、新闻、质量报告和同步任务做解释、总结和查询编排 |
| 因子实验辅助 | 让 AI 生成因子草案和实验计划，但执行必须走受控数据读取接口 |
| 结果可追溯 | 回测和 AI 生成的研究结果记录输入数据集版本、时间范围和参数 |

## 暂不进入阶段一

- 完整回测引擎。
- AI 自动交易或绕过数据治理的自主写入。
- 实盘交易。
- 多租户和复杂权限。
- 插件市场。
- tick、orderbook、实时流。
- Redis、MongoDB、Celery 作为强依赖。
- 通用新闻爬虫平台。
