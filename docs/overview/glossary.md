# 术语表

| 术语 | 含义 |
| --- | --- |
| 个人股票数据管理与研究工作台 | Quant 的产品定位，面向本地股票、行情、新闻和数值研究 |
| 工作台层 | 面向日常使用的产品层，包含总控台、股票池、股票详情、新闻汇总和数值数据 |
| 控制面 | 面向数据维护的产品层，包含数据源、同步任务、数据库状态、质量、批次和血缘 |
| 股票池 | 系统管理的 A 股股票基础资料集合，对应后端 `stocks` 领域 |
| 数据源 | AKShare、BaoStock、AData、Tushare、Stock SDK 等外部数据入口 |
| provider | 具体数据源实现，正式写入时必须记录真实 provider |
| `auto` | 自动选择模式，按启用状态、能力、健康和优先级选择 provider |
| adapter | 数据源适配器，负责 fetch、normalize、capabilities、health check |
| schema_validate | 标准化数据写入前的字段和口径校验 |
| ingest batch | 入库批次，记录任务、来源、范围、版本、写入量和质量状态 |
| data lake | 本地 Parquet 行情数据湖 |
| metadata database | 元数据库，保存股票、数据源、任务、批次、数据集、质量报告等 |
| DuckDB | 查询 Parquet 的本地分析引擎，不是主持久化数据库 |
| 数据质量 | 对数据完整性、重复、缺失、异常价格、覆盖范围等进行检查 |
| 水位线 | 某类数据在市场或股票维度上的最近成功同步位置 |
| silver 数据 | 清洗、校验后的主数据层 |
| gold 数据 | 后续给研究、因子、策略和回测直接使用的数据层 |
| 市场级日线缺口补齐 | `daily_bars_market_repair` 任务，根据股票池和交易日历补齐缺失日线 |
| 新闻文章 | 后续新闻闭环的标准化数据对象，包含标题、来源、发布时间、链接和摘要 |
| 股票新闻关联 | 新闻与股票代码的关联关系，用于股票详情和新闻汇总过滤 |
