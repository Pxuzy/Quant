## 1. 产品定位

**一句话**：本地优先的个人 A 股看盘工具，数据自持、多源可用、轻量高效。

**目标用户**：自己。用于每日看盘、看新闻，后期扩展回测和策略制定。

## 3. 功能规格

### 3.1 仪表盘 `/dashboard`

**布局（从上到下）：**

#### 3.1.1 大盘指数栏

- 展示：上证指数、深证成指、沪深300、创业板指
- 数据：现价、涨跌额、涨跌幅
- 数据源：腾讯行情 API（直连），备用 stock-sdk MCP
- 刷新：跟随全局刷新频率（默认30秒，可配置）

#### 3.1.2 板块异动栏

- 展示：按涨跌幅排序，显示涨/跌 top 板块
- 默认展示前10个，按 `change_percent` 倒序
- 数据源：腾讯板块行情 API 或 stock-sdk batch
- 刷新：跟随全局刷新频率

#### 3.1.3 自选股行情栏

- 展示：用户添加的股票列表
- 列：名称、代码、现价、涨跌额、涨跌幅、PE、PB、量比、委比
- 数据源：腾讯行情 API
- 刷新：跟随全局刷新频率
- 自选股管理：
  - 手动添加（输入代码）
  - 手动删除（点击删除按钮）
  - 数据存前端 state + localStorage（不写数据库）
  - 支持搜索股票（腾讯股票搜索 API）

#### 3.1.4 全局刷新控制

- 页面顶部或角落有刷新频率设置
- 选项：手动、10秒、30秒、60秒
- 刷新频率存 localStorage，用户偏好持久化
- 手动刷新按钮：点击立即刷新一次

---

### 3.2 股票详情 `/stock/:code`

**布局（从左到右或从上到下）：**

#### 3.2.1 K线图

- 组件：`lightweight-charts`（TradingView 开源）
- 周期：日线、周线、月线，Tab 切换
- 叠加：成交量柱状图（下方）
- 数据源：腾讯历史K线 API 或 stock-sdk
- 交互：鼠标悬停显示 OHLC + 成交量

#### 3.2.2 实时行情栏

- 展示：现价、涨跌额、涨跌幅、开盘、收盘、昨收、最高、最低
- 估值：PE、PB、流通市值、总市值
- 盘口：买1-5价量、卖1-5价量
- 数据源：stock-sdk MCP 实时行情

#### 3.2.3 技术指标

- 显示：MA5、MA10、MA20（收盘价均线）
- 来源：从K线数据计算

#### 3.2.4 新闻关联

- 展示：该股票相关新闻标题列表
- 数据源：雪球新闻 API 或 stock-sdk
- 点击跳转详情或外链

---

### 3.3 新闻中心 `/news`

#### 3.3.1 新闻源

- 源1：**雪球热门** — 质量高，有股票关联
- 源2：**百度实时热搜** — 覆盖面广
- 汇总展示，按时间倒序

#### 3.3.2 新闻卡片

- 标题、来源平台、发布时间、摘要（截取前100字）
- 点击跳转原文
- 关联股票标签（如果有）

#### 3.3.3 过滤

- 按平台筛选（全部/雪球/百度）
- 按日期筛选（今天/近3天/近7天）
- 按股票代码筛选（输入代码查相关新闻）

#### 3.3.4 数据源处理

- 并发拉取多源
- 合并去重（按标题相似度）
- 降级策略：雪球失败→只显示百度；都失败→显示错误提示

## 6. 设计决策记录

| 日期 | 决策 | 理由 |
|------|------|------|
| 2026-06-22 | 砍到5个核心页面 | 自用工具不需要复杂后台 |

## 7. 风险

| 风险 | 缓解 |
|------|------|
| 腾讯 API 限流/封禁 | stock-sdk 备用；加请求间隔 |
| 雪球 API 反爬 | 加 User-Agent；降低请求频率 |
| K线数据缺失 | 数据后台有补齐能力，引导用户去补 |
| 前端刷新频繁 | 用户可配置；交易时段自动降频 |
